There are six maxims that every programmer should follow:

- Be specific;
- Use abstractions;
- Be concise;
- Use the provided tools;
- Don't be obscure;
- Be consistent.

Using the most specific form possible makes it easier for your reader to understand  
your intent. One important way of being specific is using abstractions. Lisp provides very  
general data structures, such as lists and arrays. These can be used to implement  
specific data structures that your program will use, but you should not make the  
mistake of invoking primitive functions directly.
